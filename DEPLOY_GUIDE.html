📘 دليل نشر PMO Dashboard على Render

⚠️ متطلبات أساسية:
  • حساب على GitHub
  • حساب على Render (مجاني)
  • المشروع مرفوع على GitHub

الخطوة 1: تسجيل الدخول إلى Render

1 افتح المتصفح وادخل إلى:

https://dashboard.render.com

اضغط على "Sign up with GitHub"
[صورة: زر تسجيل الدخول بـ GitHub]

الخطوة 2: إنشاء Web Service جديد

2 بعد تسجيل الدخول، اضغط على الزر الأزرق:

"New +" → "Web Service"
[صورة: قائمة New + المنسدلة]

الخطوة 3: اختيار المستودع

3 اختر "Build and deploy from a Git repository"
ثم اضغط "Next"
[صورة: خيار Build and deploy from a Git repository]

الخطوة 4: البحث عن المستودع

4 في حقل البحث، اكتب:

pmo-dashboard-SA

ثم اضغط على "Connect" بجانب المستودع
[صورة: قائمة المستودعات مع زر Connect]

الخطوة 5: إعدادات الخدمة

5 املأ البيانات كالتالي:

الحقل القيمة
Name pmo-dashboard-sa
Region Frankfurt (EU Central)
Branch main
Runtime Node
Build Command npm install && pip install -r requirements.txt
Start Command node server.js
Plan Free
[صورة: نموذج الإعدادات مملوء]

الخطوة 6: إنشاء الخدمة

6 اضغط على الزر الأخضر:

"Create Web Service"
[صورة: زر Create Web Service]

الخطوة 7: انتظار التشغيل

7 انتظر 3-5 دقائق حتى يكتمل النشر.
ستشاهد سجلات البناء في الوقت الفعلي.
[صورة: شاشة Logs showing build progress]
✅ عند اكتمال النشر:
  • سيظهر زر أخضر بجانب اسم الخدمة
  • الرابط سيكون: https://pmo-dashboard-sa.onrender.com
  • اضغط على الرابط لفتح التطبيق

🔧 في حالة وجود مشاكل:

! إعادة النشر:
  1. ادخل إلى خدمتك في Render
  2. اضغط على "Manual Deploy"
  3. اختر "Deploy latest commit"
[صورة: زر Manual Deploy]

📱 بيانات الدخول:

اسم المستخدم admin
كلمة المرور pmo2024

⚠️ ملاحظات مهمة:
  • الخطة المجانية تدخل في وضع السكون بعد 15 دقيقة من عدم الاستخدام
  • عند أول زيارة بعد السكون، يستغرق 30-60 ثانية للاستيقاظ
  • البيانات تُحفظ في قاعدة SQLite مؤقتاً (قد تُحذف عند إعادة التشغيل)
